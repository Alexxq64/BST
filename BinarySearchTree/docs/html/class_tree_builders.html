Рекурсивная функция для построения ИСДП

Рекурсивная функция построения ИСДП

Аргументы
sortedDataОтсортированный массив
startНачальный индекс подмассива
endКонечный индекс подмассива
Возвращает
Корень поддерева
Аргументы
sortedDataОтсортированный массив данных
startНачальный индекс подмассива
endКонечный индекс подмассива
Возвращает
Указатель на корень построенного поддерева

Алгоритм:

  1. Если start > end, возвращаем nullptr (пустое поддерево)
  2. Вычисляем средний индекс: mid = start + (end - start) / 2
  3. Создаем узел с ключом sortedData[mid]
  4. Рекурсивно строим левое поддерево из элементов [start, mid-1]
  5. Рекурсивно строим правое поддерево из элементов [mid+1, end]
Заметки
Гарантирует сбалансированность: разница высот поддеревьев ≤ 1

Построение B-дерева (ДБД).

Аргументы
dataМассив уникальных чисел
tМинимальная степень узла (t >= 2)
Возвращает
Указатель на корень ДБД
Аргументы
dataМассив ключей для вставки
tМинимальный порядок B-дерева (t ≥ 2)
Возвращает
Корень построенного B-дерева

Строит B-дерево заданного порядка t путем последовательной вставки ключей. B-дерево сохраняет балансировку автоматически за счет свойств:

  • Каждый узел содержит от t-1 до 2t-1 ключей (кроме корня)
  • Все листья находятся на одном уровне
Заметки
Для t=2 получаем 2-3 дерево

In-order обход B-дерева

Аргументы
nodeТекущий узел
keysВектор для сохранения ключей в отсортированном порядке

Выполняет обход B-дерева, возвращая ключи в возрастающем порядке. Алгоритм:

  1. Рекурсивно обходит все дочерние узлы
  2. После каждого дочернего узла добавляет соответствующий ключ
  3. Для листовых узлов добавляет все ключи
Заметки
Результат всегда отсортирован для корректного B-дерева

Разделение дочернего узла при переполнении

Разделение переполненного дочернего узла

Аргументы
parentРодительский узел
indexИндекс дочернего узла в массиве детей
tМинимальная степень узла
parentРодительский узел
indexИндекс дочернего узла для разделения
tМинимальный порядок дерева

Разделяет переполненный узел y на два узла (y и z) и поднимает средний ключ в родительский узел. Алгоритм:

  1. Создает новый узел z
  2. Переносит последние t-1 ключей из y в z
  3. Переносит последние t дочерних узлов из y в z (если y не лист)
  4. Вставляет средний ключ y в parent
  5. Добавляет z как дочерний узел parent
